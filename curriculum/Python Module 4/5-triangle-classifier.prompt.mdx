# Triangle Classifier

Time to combine everything from this module: comparisons, logical operators, an ordered `elif` chain, and a validity check that has to happen before anything else. Given three side lengths, you'll decide whether they even form a triangle, and if so, what kind.

## The rules

First, validity. Three lengths form a valid triangle only if every side is positive **and** the sum of any two sides is greater than the third. That last part is the *triangle inequality*: for sides `a`, `b`, `c`, you need `a + b > c` and `a + c > b` and `b + c > a`.

If the sides are valid, classify by how many are equal:

- all three equal: `"equilateral"`
- exactly two equal: `"isosceles"`
- none equal: `"scalene"`

## Your task

Write a function `classify_triangle(a, b, c)` that returns:

- `"invalid"` if the three lengths don't form a triangle,
- otherwise `"equilateral"`, `"isosceles"`, or `"scalene"`.

## Requirements

- Define a function named `classify_triangle` taking three arguments, `a`, `b`, `c`.
- It must **return** the classification string, not print it.
- Check validity first. If the sides are invalid, return `"invalid"` and don't try to classify.
- A side of `0` or a negative side makes the triangle invalid.
- Then use an `if`/`elif`/`else` chain for the three valid shapes.

## Example

```python
print(classify_triangle(3, 3, 3))   # equilateral
print(classify_triangle(5, 5, 8))   # isosceles
print(classify_triangle(3, 4, 5))   # scalene
print(classify_triangle(1, 1, 5))   # invalid — 1 + 1 is not greater than 5
print(classify_triangle(0, 4, 5))   # invalid — a side can't be zero
```
