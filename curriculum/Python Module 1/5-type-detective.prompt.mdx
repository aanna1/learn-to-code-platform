# Type Detective

Python's four foundational types look different but behave very differently. A number inside quotes isn't a number — it's text that happens to contain digits. This exercise trains you to tell them apart.

## Your task

Complete the function `identify_type(value)` so that it returns a string naming the type of `value`:

| If `value` is... | Return |
|---|---|
| A string (`str`) | `"string"` |
| A whole number (`int`) | `"integer"` |
| A decimal number (`float`) | `"float"` |
| `True` or `False` (`bool`) | `"boolean"` |

## Requirements

- Return the strings exactly as shown in the table above (lowercase, no quotes in the return value itself).
- Do not use `if/else` — use `type()` and comparisons. The function body should be a single return statement.
- The function must handle all four types.

## Example

```python
identify_type("hello")   # returns "string"
identify_type(42)        # returns "integer"
identify_type(3.14)      # returns "float"
identify_type(True)      # returns "boolean"
identify_type("99")      # returns "string"  — it's in quotes, so it's text
```

## Important edge case

`True` and `False` are booleans, but `bool` is actually a subtype of `int` in Python. Check for `bool` *before* checking for `int`, or `True` will incorrectly return `"integer"`.
