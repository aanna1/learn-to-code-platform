You have a `TextProcessor` class that cleans, transforms, and validates text. Your job is to write a complete test suite for it.

```python
class TextProcessor:
    def __init__(self, text: str):
        """Raise ValueError if text is empty or whitespace-only."""

    def word_count(self) -> int:
        """Return the number of words (split on whitespace)."""

    def uppercase(self) -> str:
        """Return the text uppercased."""

    def truncate(self, max_len: int) -> str:
        """Return text trimmed to max_len characters.
        Raise ValueError if max_len < 1."""

    def contains(self, substring: str) -> bool:
        """Return True if substring appears in text (case-sensitive)."""
```

Write **six** test functions that together cover the class thoroughly:

- `test_empty_text_raises` — creating `TextProcessor("")` raises `ValueError`
- `test_whitespace_text_raises` — creating `TextProcessor("   ")` raises `ValueError`
- `test_word_count` — a simple sentence returns the correct word count
- `test_uppercase` — the text is fully uppercased
- `test_truncate` — truncating to fewer characters than the text length works correctly
- `test_truncate_invalid_max_len` — `truncate(0)` raises `ValueError`

Use `make_processor(text)` (already defined) as your fixture. Write clear `assert` messages so a failing test explains what went wrong.
