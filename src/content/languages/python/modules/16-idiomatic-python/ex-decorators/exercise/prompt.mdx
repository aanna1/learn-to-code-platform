# Decorators

A decorator takes a function and returns a (usually wrapped) function. The `@` syntax is just shorthand for `f = decorator(f)`. This exercise has two parts.

## Part 1: `log_calls`

Implement `log_calls` as a decorator that wraps any function and logs each call. For a function `add(2, 3)` that returns `5`, it should log:

```
add(2, 3) -> 5
```

Requirements:

- Use `logging.getLogger(__name__)` and log at the `INFO` level.
- The log message format must be `"{name}({args}) -> {result}"` where `args` is a comma-separated `repr` of each positional argument.
- Use `@functools.wraps` so the wrapper preserves the original function's name and docstring.
- Keyword arguments do not need to appear in the log message (positional only is fine).

## Part 2: `repeat(times)`

Implement `repeat` as a decorator factory — a function that takes a configuration argument and returns a decorator.

```python
@repeat(times=3)
def greet(name: str) -> str:
    print(f"Hello, {name}!")
    return name
```

Calling `greet("world")` should call the original function `times` times and return the last result.

Requirements:

- `repeat` takes one keyword argument: `times` (a positive integer).
- The wrapper must call the original function exactly `times` times.
- Use `@functools.wraps` on the inner wrapper.
- The return value must be the result of the final call.

## Checking your work

```python
import logging
logging.basicConfig(level=logging.INFO, format="%(message)s")

@log_calls
def add(a, b):
    """Add two numbers."""
    return a + b

add(2, 3)   # logs: add(2, 3) -> 5
print(add.__name__)    # add
print(add.__doc__)     # Add two numbers.

@repeat(times=3)
def greet(name):
    print(f"Hello, {name}!")
    return name

result = greet("world")   # prints Hello, world! three times
print(result)             # world
```
