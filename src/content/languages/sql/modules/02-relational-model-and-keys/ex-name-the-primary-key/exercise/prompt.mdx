The lecture's big idea: every table needs a **primary key** — one column guaranteed unique for every row — so no two entries can ever be confused. This `students` table has one, `student_id`, and it earns its keep here. Look at rows 1 and 4: two different students both named Jack, both studying Biology. On name and major alone, they're indistinguishable.

The starter query returns only `name` and `major`, so the two Jacks come back as two identical rows. The one column that can tell them apart is the primary key.
